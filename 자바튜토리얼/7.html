※상속 <상속의 개념> ●상속이란? - 두 클래스를 조상과 자손으로 관계를 맺어주는 것. - 자손은 조상의 모든 멤버를 상속 받는다. ●상속의 장점 -상속을 통하여 기존 클래스의 필드와 메소드를 재사용 -기존 클래스의 일부 변경도 가능 -상속을 이용하게 되면 복잡한 GUI 프로그램을 순식간에 작성 -상속은 이미 작성된 검증된 소프트웨어를 재사용 -신뢰성 있는 소프트웨어를 손쉽게 개발, 유지 보수 -코드의 중복을 줄일 수 있다. ●부모 클래스와 자식 클래스 -부모 클래스는 추상적이고 자식 클래스는 구체적이다. 부모클래스 자식클래스 동물 사자, 개, 고양이 탈것 자동차, 버스, 트럭 학생 대학원생, 학부생 ●상속의 형식 class Childclass extends Parentclass { ⤷자식클래스 또는 ⤷부모클래스 또는 서브클래스라 한다. 수퍼클래스라 한다. ... } -자식 클래스는 부모 클래스를 포함한다. ●상속받은 필드와 메소드 사용 -멤버 연산자(.)를 사용한다. ●상속이 필요한 이유 -코드를 재사용할 수 있다. -중복을 줄일 수 있다. car{ Truck{ Bus{ setSpeed() setSpeed() setSpeed() turn() turn() turn() →각 클래스에 코드가 중복된다. } } } 상속후↓ Vehicle{ setSpeed() →중복되는 코드는 부모 클래스에 모은다. turn() } Car{ } Truck{ } Bus{ } →자식클래스 <상속과 접근 제어> ●상속과 접근제어 -자식 클래스에서는 부모 클래스의 필드 중에서 private로 선언된 것과 다른 패키지의 default만 제외하고 모두 접근할 수 있다. ●접근 제어 지정자 접근지정자 클래스 패키지 자식클래스 전체 세계 public O O O O protected O O O X default(없음) O O X X private O X X X -default와 protected 의 차이 -default는 같은 패키지라면 상속이 가능하지만, 다른 패키지라면 상속이 불가능하다. -protected는 같은 패키지든 다른 패키지든 상관없이 상속이 가능하다. <메소드 오버라이딩> ●메소드 오버라이딩 -메소드 오버라이딩(method overriding): 자식 클래스가 필요에 따라 상속된 메소드를 다시 정의하는 것 -부모 클래스의 메소드를 자식 클래스가 자신의 필요에 맞추어서 변경하는 것이다. ●메소드를 오버라이딩하려면 -메소드의 이름, 반환형, 매개 변수의 개수와 데이터 타입이 일치하여야 한다. public class Animal { public void sound() { } }; ↓오버라이딩이 아님 public class Dog extends Animal { public int sound() { } }; ●super 키워드 super를 사용하여 부모 클래스 멤버 접근 <상속과 생성자> ●상속과 생성자 -서브 클래스의 객체가 생성될 때, 서브 클래스의 생성자만 호출될까? 아니면 수퍼 클래스의 생성자도 호출되는가? class Base{ public Base(String msg) { System.out .println( "Base() 생성자" ); } } class Derived extends Base { public Derived() { System.out .println( "Derived() 생성자" ); } } public class Test { public static void main(String[] args) { Derived r = new Derived(); } } 실행결과: Base() 생성자 →Derived객체를 생성했는데 Base생성자까지 호출되었다. Derived() 생성자 -왜 Derived객체를 생성했는데 Base생성자까지 호출될까? -자식클래스 객체는 부모클래스에서 상속된 부분과 자식클래스가 추가한 부분이 합쳐져 있다고 생각하여야 한다. -생성자의 호출 순서: (부모 클래스의 생성자) → (자식 클래스의 생성자) 순 ●부모클래스 생성자 호출방식 -명시적인 호출 -묵시적인 호출 ●명시적인 생성자 호출 -super를 이용하여서 명시적으로 수퍼 클래스의 생성자 호출 class Shape { public Shape(String msg) { System.out.println("Shape 생성자() " + msg); } }; public class Rectangle extends Shape { public Rectangle(){ super("from Rectangle"); // 명시적인 호출 System.out.println("Rectangle 생성자()"); } }; public class Test { public static void main (String[] args) { Rectangle r = new Rectangle(); } }; 실행결과: Shape 생성자() from Rectangle Rectangle 생성자() ●묵시적인 생성자 호출 -명시적으로 부모클래스의 생성자를 호출해주지 않아도 자식클래스의 객체가 생성될 때 자동적으로 부모클래스의 매개 변수가 없는 생성자가 호출된다. class Shape { public Shape() { System.out.println("Shape 생성자()"); } }; class Rectangle extends Shape { public Rectangle() { //super을 쓰지 않아도 자동적으로 부모클래스의 매개 변수가 없는 생성자가 호출 System.out.println("Rectangle 생성자()"); } }; public class Test { public static void main (String[] args) { Rectangle r = new Rectangle(); } }; 실행결과: Shape 생성자() Rectangle 생성자() <추상 클래스> ●추상 클래스(abstract class) -추상 클래스(abstract class): 몸체가 구현되지 않은 메소드를 가지고 있는 클래스 -추상 클래스는 추상적인 개념을 표현하는데 적당하다. EX) Animal (추상 클래스) ⊃ Lion, GoldFish, Dog, Cat ●추상 클래스의 정의 -추상 클래스를 만들기 위해서는 클래스 선언 시에 앞에 abstract를 붙인다. -추상 클래스는 하나 이상의 추상 메소드를 가지고 있다. 추상 메소드란 아래 예시의 move()처럼 몸체가 없는 메소드를 말한다. public abstract class Animal { //추상 클래스 정의 public abtract void move(); //추상 메소드 정의 ... }; ●추상 메소드의 예 -EX) Shap(추상 클래스) ⊃ Rectangle, Circle, Triangle abstract class Shape { //추상 클래스 Shape를 선언한다. 추상 클래스로는 객체를 생성할 수 없다. private int x, y; public void move(int x, int y){ //추상 클래스라고 하더라도 추상 메소드가 아닌 보통의 메소드도 가질 수 있다. this.x = x; this.y = y; } public abstract void draw(); /*추상 메소드를 선언한다. 추상 메소드를 하나라도 가지면 추상 클래스가 된다. 추상 메소드를 가지고 있는데도 abstract를 class앞에 붙이지 않으면 컴파일 오류가 발생한다.*/ }; class Rectangle extends Shape { private int width, height; public void draw() { // 추상 메소드 구현, 자식클래스에서 추상 메소드를 구현하지 않으면 컴파일 오류가 발생한다. System.out .println("원 그리기 메소드"); } }; <상속과 다형성> ●다형성이란? -다형성(polymorphism)이란 객체들의 타입이 다르면 똑같은 메시지가 전달되더라도 서로 다른 동작을 하는 것 -다형성은 동일한 코드로 다양한 타입의 객체를 처리할 수 있는 기법이다. ●자바에서의 자료형 검사는 엄격하다! -클래스 A의 참조 변수로 클래스 B의 객체를 참조할 수는 없다. class A { A() { } } class B { B() { } } public class TypeTest1 { public static void main(String args[]) { A a = new B(); // NO! } } ●그러나 예외가 있다!
